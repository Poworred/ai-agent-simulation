# AI 南大｜多智能体校园社会模拟系统项目总览

## 1. 一句话定位

AI 南大是一个面向中国高校场景的轻量级多智能体社会模拟原型。它参考 HKUST Aivilization、Stanford Generative Agents 和 AI Town 的思路，把南昌大学校园抽象为一个 AI Agent 可以感知、行动、对话、记忆和反思的虚拟环境。

第一版不是要复刻港科大 10 万 Agent 规模的平台，而是要做一个 5-20 个 Agent 的网页 MVP，用于验证一个更小但更清晰的问题：

> AI 新生在南大生活一周，这件事是否有可看性、传播性，以及作为 Agent 项目的技术展示价值？

## 2. 为什么做这个项目

这个项目的价值不在于“又做一个 AI 聊天角色”，而在于把 AI Agent 放进一个公共世界里。角色不是只在聊天框里回复用户，而是会在时间、地点、课程、社团活动、关系网络和随机事件的约束下行动。

它可以展示三类能力：

- Agent 系统能力：人格、目标、计划、环境感知、行动选择、对话、记忆和反思。
- 产品抽象能力：把真实校园生活拆成地点、日程、事件、关系和状态。
- 研究型产品能力：参考前沿项目和论文，做出可验证、可演示、可迭代的本土化原型。

对作品集和简历而言，这个项目的理想表达不是“复刻 AI 小镇”，而是：

> 参考 Aivilization 和 Generative Agents 的多智能体社会模拟范式，设计并实现一个面向中国高校校园场景的低成本 Agent 原型。

## 3. 参考项目

本项目主要参考以下方向：

- HKUST Aivilization：大规模 AI 社会模拟、公民科学、AI 居民、经济与文化规范、人类干预。
- Stanford Generative Agents：记忆、反思、计划、观察、行动，以及 Smallville 里的 25 个 Agent。
- AI Town：可部署的网页小镇原型，强调 AI 角色在虚拟空间中生活、聊天和社交。
- Project Sid：多 Agent 在 Minecraft 环境中的文明化叙事，提供大规模 Agent 社会的想象空间。
- Concordia：用 Game Master 和组件化 Agent 架构组织社会模拟。
- AgentSociety：大规模城市社会模拟与社会实验，提醒本项目关注验证和边界。

## 4. 第一版目标

第一版只做轻量 MVP，目标是把核心链路跑通：

- 用户打开网页后可以看到一个简化南大校园地图。
- 地图上有 5-20 个 AI 角色在不同地点行动。
- 角色有身份、人设、课表、目标、关系和记忆。
- 系统按时间推进一周校园生活。
- Agent 会上课、吃饭、参加社团、迷路、聊天、遇到随机事件。
- 用户可以点开角色查看状态、事件流、关系和记忆摘要。
- 用户可以对少数角色进行轻量干预，例如提醒、建议、给任务。

## 5. 第一版不做什么

为了保证项目能落地，MVP 明确不做以下内容：

- 不接入真实教务系统。
- 不接入真实用户账号体系。
- 不做真实社交平台。
- 不做真钱经济或交易系统。
- 不做 100+ Agent 或 10 万 Agent 级规模。
- 不做完整 GIS 校园地图。
- 不宣称具备真实社会科学预测能力。
- 不把“看起来像人”当成“真实模拟人”。

## 6. 项目成果形态

最终希望形成三类成果：

- 可演示网页：能够展示地图、Agent、事件流和角色详情。
- 项目文档：PRD、架构说明、参考论文、MVP 拆解、用户测试记录。
- 简历表达：能够压缩成一个清楚的个人 Agent 项目，证明产品抽象、Agent 架构和工程实现能力。
