# AI 南大项目记忆

这份文件用于后续快速恢复上下文。以后继续讨论这个项目时，可以先读这里，避免重新解释。

## 1. 当前项目定位

项目名称暂定：

> AI 南大｜多智能体校园社会模拟系统

项目本质：

> 一个参考 HKUST Aivilization、Stanford Generative Agents 和 AI Town 的中国高校场景轻量化 Agent 原型。

它不是传统游戏项目，也不是纯聊天机器人项目，而是一个研究型产品原型。核心能力是把校园生活转译成 AI Agent 可以运行的世界状态。

## 2. 已确定的关键判断

- 这是作品集/简历项目，优先证明能力，而不是一开始追求商业化。
- 不写成“复刻港科大 10 万 Agent”，而写成“参考港科大方向的本土化轻量 Agent 原型”。
- 第一版 Agent 数量控制在 5-20 个。
- 第一版使用网页轻量 MVP，不上来做大型游戏客户端。
- 核心叙事是“AI 新生在南大生活一周”。
- 第一版不接入真实教务系统，不采集真实隐私数据。
- 项目要同时体现技术、产品、策划和研究调研能力。
- 项目最终需要能写进简历，而不是只停留在脑暴。

## 3. 核心叙事

核心故事不是“校园元宇宙”，而是：

> 一批 AI 新生来到南昌大学，在一周内经历上课、吃饭、迷路、社团招新、失物招领、社交关系和突发事件。用户观察他们如何适应大学生活，也可以对他们进行有限干预。

这个叙事比“大规模 AI 社会”更适合个人项目，因为它更具体、更容易做出 demo，也更容易产生校园传播点。

## 4. 项目要证明什么

对外展示时，项目要证明：

- 我理解多智能体社会模拟的基本范式。
- 我能把论文中的 memory、reflection、planning、observation 等概念转成产品模块。
- 我能把真实校园生活拆成地点、时间、角色、关系、事件和状态。
- 我知道如何用低成本策略避免每个 NPC 全时调用大模型。
- 我有边界意识，不会把“看起来像人”夸大成“真实模拟人”。

## 5. 最重要的技术主线

Agent 行为循环：

```text
人格设定 -> 目标生成 -> 日程规划 -> 环境感知 -> 行动选择 -> 对话生成 -> 记忆写入 -> 事件反思
```

低成本策略：

```text
规则处理日常行为
LLM 处理关键事件
摘要压缩长期记忆
缓存固定文本
事件触发而非全时推理
```

系统数据流：

```text
World State -> Agent Perception -> Planner -> Action -> Environment Update -> Memory Write -> Event Feed
```

## 6. 重要边界

MVP 不做：

- 真实教务系统。
- 真实用户账号。
- 真实用户社交。
- 钱包、交易、真钱经济。
- 完整 GIS 地图。
- 100+ Agent 或 10 万 Agent。
- 复杂政治、宗教、敏感社会议题。
- 真实社会科学预测。

## 7. 当前最合理的下一步

下一步不是继续扩大概念，而是做一个最小可演示原型：

- 手工设定 8 个 Agent。
- 手工整理 6-8 个校园地点。
- 手工设计一周课表和活动。
- 先用规则跑出基础行为。
- 再把 LLM 接入对话、反思和复杂事件。
- 最后做网页展示地图、事件流和角色详情。

## 8. 简历表达原则

不要写：

> 复刻港科大 AI 小镇，实现大规模 AI 文明。

推荐写：

> 参考 HKUST Aivilization、Stanford Generative Agents 与 AI Town，设计并实现一个面向中国高校场景的轻量级多智能体社会模拟原型。

核心表达：

> 将南昌大学校园抽象为 Agent 可感知、可行动的虚拟环境，构建人格、目标、日程、感知、行动、对话、记忆和反思组成的 Agent 行为循环，并通过网页端展示 AI 新生一周校园生活。
