# AI 南大简历表达

## 1. 推荐项目名称

中文：

> AI 南大｜多智能体校园社会模拟系统

英文：

> AI Campus Agent Simulation

更偏技术的写法：

> LLM Agent Campus Simulation｜基于 LLM Agent 的校园社会模拟系统

## 2. 一句话项目描述

> 参考 HKUST Aivilization、Stanford Generative Agents 与 AI Town，设计并实现一个面向中国高校场景的轻量级多智能体社会模拟原型，将南昌大学校园抽象为 Agent 可感知、可行动、可对话、可记忆的虚拟环境。

## 3. 进行中版本

适用于项目还处在调研、PRD、架构设计或早期原型阶段。

**AI 南大｜多智能体校园社会模拟系统**  
个人项目｜LLM Agent / Multi-Agent Simulation / Web Prototype

- 调研 HKUST Aivilization、Stanford Generative Agents、AI Town、Project Sid 等多智能体社会模拟项目，梳理 Agent 记忆、计划、反思、行动与环境建模机制。
- 基于南昌大学校园场景，设计“AI 新生在南大生活一周”的轻量级网页原型，将校园地图、课表、社团活动、失物招领和随机事件抽象为 Agent 可感知、可行动的世界状态。
- 规划 Agent 行为循环，包括人格设定、目标生成、日程规划、环境感知、行动选择、多轮对话、记忆写入与事件反思。
- 设计低成本模拟策略，通过规则驱动日常行为、事件触发 LLM 调用、短上下文摘要和缓存机制，避免每个 NPC 全时调用大模型。
- 输出 PRD、参考项目分析、技术架构和 MVP 任务拆解，为后续原型开发和用户测试提供依据。

## 4. MVP 完成后版本

适用于已经做出可演示网页后。

**AI 南大｜多智能体校园社会模拟系统**  
个人项目｜LLM Agent / Multi-Agent Simulation / Web Game

- 参考 HKUST Aivilization、Stanford Generative Agents 与 AI Town，实现一个面向中国高校场景的轻量级多智能体校园模拟原型，支持 5-20 个 AI Agent 在虚拟校园中自主行动。
- 将南昌大学校园抽象为 Agent 可感知、可行动的世界状态，包括地点、课表、社团活动、随机事件、角色关系与记忆系统。
- 构建 Agent 行为循环，支持人格设定、目标生成、日程规划、环境感知、行动选择、多轮对话、记忆写入与每日反思。
- 实现低成本运行策略，使用规则驱动常规行动，仅在关键事件、对话和反思阶段调用 LLM，并通过摘要、缓存和事件触发控制 token 成本。
- 开发网页演示界面，展示校园地图、Agent 状态、全局事件流和角色详情，并通过小范围用户测试验证项目的可看性与传播性。

## 5. 更偏 AI 产品岗的表达

如果投 AI 产品、Agent 产品、AIGC 产品方向，可以这样写：

- 面向校园社交与社团纳新场景，设计一款 LLM Agent 驱动的轻量网页互动产品，用户可观察 AI 新生在校园中上课、社交、参加活动并触发随机事件。
- 将真实校园生活拆解为地图、时间、角色、关系、记忆和事件系统，形成可运行的 Agent 产品框架。
- 对比 Aivilization、AI Town、Character.AI 等项目，明确本项目差异：从一对一角色聊天转向有公共世界、有行动后果、有关系演化的校园社会模拟。

## 6. 更偏技术岗的表达

如果投前端、后端、Agent 工程方向，可以这样写：

- 设计多 Agent 调度架构，以 World State 为中心组织地点、事件、角色状态和记忆数据。
- 实现 Agent Runtime，将人格、日程、感知、行动、对话和反思拆成可组合模块。
- 封装 LLM 调用层，对复杂事件决策、对话生成和记忆摘要进行结构化输出校验。
- 通过规则引擎、事件触发、缓存和短上下文摘要降低推理成本，提高演示稳定性。

## 7. 不建议写法

不要写：

> 复刻港科大 AI 小镇，实现大规模 AI 文明。

问题：

- 容易被追问规模、成本、数据、评估和社会结构。
- 听起来像蹭热点，不像个人项目。
- 如果没有真实大规模运行，会显得夸大。

不要写：

> 做了一个 AI 社交游戏。

问题：

- 太泛，无法体现 Agent 技术含量。
- 看不出你参考了前沿项目和论文。
- 看不出产品抽象能力。

推荐写：

> 参考 HKUST Aivilization 的多智能体社会模拟方向，做了一个面向中国高校校园场景的本土化轻量 Agent 原型。

## 8. 面试时可以怎么讲

可以用 30 秒这样介绍：

> 这个项目的出发点是，我不想只做一个 AI 聊天角色，而是想验证 AI Agent 放进公共校园世界后，会不会产生更有意思的行为和叙事。所以我参考了 Aivilization、Generative Agents 和 AI Town，把南昌大学校园抽象成地点、课表、事件、关系和记忆系统，让一批 AI 新生在里面生活一周。第一版重点不是规模，而是把 Agent 的人格、计划、感知、行动、对话、记忆和反思跑通，并通过网页把过程展示出来。
