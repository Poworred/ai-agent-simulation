# AI 南大参考项目与论文

## 1. HKUST Aivilization

参考链接：

- HKUST 官方新闻：https://hkust.edu.hk/news/hkust-launches-worlds-largest-ai-powered-educational-sandbox-game-advancing-ai-literacy-and
- AIvilization v0：https://arxiv.org/abs/2602.10429

Aivilization 是港科大推出的大规模 AI 多智能体社会模拟沙盒。官方叙事强调 AI 居民在虚拟社会中发展治理结构、经济体系和文化规范。论文版 AIvilization v0 重点讨论资源约束经济、统一 Agent 架构、长期目标、adaptive agent profile、分层规划、人类干预和长期记忆。

对 AI 南大的启发：

- 不要只做聊天 NPC，要让 Agent 生活在公共世界里。
- 人类干预不应该是直接改写 Agent，而应该通过记忆、目标或环境事件影响 Agent。
- 大规模不是第一版关键，关键是 Agent 是否能在约束环境中持续行动。
- 资源、时间、教育和职业路径可以启发校园里的课程、社团、成长路线。

本项目不照搬的部分：

- 不做 10 万 Agent。
- 不做复杂经济系统。
- 不宣称形成真实社会治理或社会科学结论。

## 2. Stanford Generative Agents

参考链接：

- 论文：https://arxiv.org/abs/2304.03442
- GitHub：https://github.com/joonspk-research/generative_agents

Generative Agents 是 AI 小镇方向的经典论文。它在 Smallville 中放置 25 个生成式 Agent，展示角色如何观察环境、写入记忆、进行反思、规划日程并与其他角色互动。情人节派对案例说明，一个用户植入的高层意图可以通过 Agent 之间的互动逐步传播。

对 AI 南大的启发：

- Agent 行为可信度来自 observation、planning、reflection 的组合。
- 记忆不是聊天记录，而是行为决策的上下文资产。
- 事件传播比单点聊天更有展示价值。
- 25 个 Agent 以内也可以产生有趣社会行为，不必一开始追求大规模。

本项目可直接借鉴：

- 记忆流。
- 反思机制。
- 日程规划。
- 角色之间自然触发对话。
- 用户通过自然语言影响某个 Agent。

## 3. AI Town

参考链接：

- GitHub：https://github.com/a16z-infra/ai-town

AI Town 是一个开源网页小镇 starter kit，强调“AI characters live, chat and socialize”。它的重要性在于把 Generative Agents 的研究范式变成更容易部署和定制的网页项目。

对 AI 南大的启发：

- MVP 应该优先做成网页，而不是重型客户端。
- 地图、角色移动、聊天气泡和事件流足以支撑第一版体验。
- 开源 starter kit 的价值是给出工程组织方式，而不是直接解决内容设计。

本项目可借鉴：

- 网页地图交互。
- 角色状态展示。
- 简化实时模拟。
- 可部署 demo 的项目形态。

## 4. Project Sid

参考链接：

- 论文：https://arxiv.org/abs/2411.00114

Project Sid 展示了多 Agent 在 Minecraft 环境中形成角色分工、规则、文化传播和文明化叙事的可能。它更偏“大规模、多 Agent、复杂环境”的想象。

对 AI 南大的启发：

- 环境越明确，Agent 行为越容易被用户理解。
- 角色分工和规则演化是强传播点。
- 大规模叙事有吸引力，但个人项目应该抽取机制，而不是照搬规模。

本项目可借鉴：

- 角色分工。
- 群体事件。
- 规则被 Agent 学习和遵守。
- 用可视化环境承载 Agent 行为。

## 5. Concordia

参考链接：

- 论文：https://arxiv.org/abs/2312.03664

Concordia 是 Google DeepMind 的生成式社会模拟框架。它用 Game Master 管理世界状态和行动仲裁，Agent 通过组件系统组织记忆、目标、观察和行动。

对 AI 南大的启发：

- 可以把系统设计成“Agent + Game Master + World State”。
- Agent 不应该随便改世界，世界状态需要统一仲裁。
- 自然语言行动需要被系统翻译成合法状态变化。

本项目可借鉴：

- Game Master 作为环境裁判。
- 组件化 Agent 结构。
- 行动合法性校验。
- 世界状态和角色状态分离。

## 6. AgentSociety

参考链接：

- 论文：https://arxiv.org/abs/2502.08691

AgentSociety 关注大规模 LLM-driven generative agents 对人类行为和社会现象的模拟。它提醒我们，多 Agent 系统不只是有趣，还会遇到验证、校准和解释问题。

对 AI 南大的启发：

- 不要把“像人”直接等同于“真实有效”。
- 用户测试和行为评估必须写进项目。
- 如果未来要走研究型产品路线，需要建立验证意识。

本项目可借鉴：

- 大规模模拟的架构视角。
- 社会实验的变量意识。
- 对行为输出做评估，而不是只看生成文本是否好玩。

## 7. 深度调研后的实现结论

2026-06-21 对 HKUST AIvilization/Aivilization 与 Stanford Generative Agents 做了一轮多来源交叉调研，主要参考一手资料：Stanford 论文、官方 GitHub、Reverie demo、AIvilization arXiv 预印本和官方 GitBook。结论如下：

### 7.1 Stanford 更适合作为 MVP 工程参考

Stanford Generative Agents 是小规模、可运行、可回放的研究原型。它的工程形态不是重型在线游戏服务，而是环境/前端服务器与 Agent simulation 后端分离运行。它的核心机制可以转译为：

```text
观察校园事件 -> 写入 memory stream -> 检索相关记忆 -> 反思 -> 计划 -> 行动 -> 更新世界
```

对 AI 南大第一版的直接启发：

- 优先做 5-8 个 Agent 的小规模校园模拟，不追求大规模。
- 事件流不仅是 UI 文案，也应该是 replay、debug 和记忆写入的基础。
- Agent 详情页至少展示当前行动、地点、对话/事件、状态和记忆。
- 种子角色和历史记忆应可控初始化，未来可以支持 JSON/CSV 导入 persona、history 和 memory stream。

### 7.2 Aivilization 更适合作为长期设计理念参考

AIvilization v0 把自己定位为公开部署的大规模人工社会，强调统一 LLM-Agent 架构、资源受限沙盒经济、层级 branch-thinking、双过程记忆、人类干预和行动可行性验证。公开资料能支持其架构理念，但工程细节、真实在线规模和长期稳定性没有充分公开验证。

对 AI 南大的可借鉴点：

- 不要把 Agent 做成单段 prompt；要拆成 profile、目标、记忆、计划、行动和反馈。
- 人类干预应进入目标、记忆或计划，而不是直接改写世界状态。
- 长期目标应拆为周目标、日目标、即时行动。
- 短期执行轨迹和长期身份画像要分开，避免短期噪声污染长期人格。
- LLM 输出动作前，需要有行动验证器/世界仲裁者，检查地点、时间、课表、关系和资源约束。

### 7.3 对 AI 南大 MVP 的最终路线判断

最稳妥路线是：

```text
先做 Stanford 式小规模校园可回放模拟
  -> 再逐步吸收 AIvilization 的资源/制度约束和行动验证机制
```

第一版应坚持：

- Next.js 前端观察界面 + FastAPI Agent simulation 后端。
- SQLite 保存 run、event、memory、relationship 和 LLM call。
- Game Master 作为行动验证器和世界仲裁者。
- memory stream + retrieval + daily reflection 作为 Agent 连续性的核心。
- 用户干预通过 intervention memory 影响下一轮计划。
- 不做 10 万 Agent、复杂经济系统、MMO 后台或社会科学预测。

## 8. 与 Character.AI 类产品的区别

Character.AI、Talkie 等产品主要是 AI 角色聊天或陪伴。它们的优势是低门槛、角色丰富、用户打开就能聊。它们的局限是大多数体验停留在一对一对话，缺少公共世界、地点约束、行动后果和角色之间的社会关系。

AI 南大的差异点是：

- 不是单个角色聊天，而是多个 Agent 同处一个校园世界。
- 不是只生成对话，而是生成行动、事件、关系和记忆。
- 用户不是只和一个角色互动，而是观察一个小型校园社会。

## 9. 参考资料如何进入项目

这些资料不只是“引用”，而要转化成项目模块：

- Generative Agents -> 记忆、反思、计划。
- Aivilization -> 长期目标、人类干预、资源约束。
- AI Town -> 网页小镇和可部署 demo。
- Concordia -> Game Master 和环境仲裁。
- Project Sid -> 角色分工和群体叙事。
- AgentSociety -> 验证意识和边界声明。
